03 — 工具系统

智能体的双手

工具系统是 Claude Code 与外部世界交互的核心能力。从文件读写到 Shell 命令，每个工具都经过严格的生命周期管理。

1

定义 — Tool Definition

通过 buildTool() 工厂创建工具，声明 inputSchema（Zod）、描述、权限检查和执行逻辑。每个工具一个独立目录。

2

注册 — Registration

核心工具在 tools.ts 中注册，MCP 工具在服务器连接时动态生成，插件工具通过 pluginLoader 注入。三者最终通过 useMergedTools 合并。

3

验证 — Input Validation

执行前使用 Zod Schema 验证输入参数，通过 classifyToolError 对错误进行分类处理。

4

权限 — Permission Check

调用 CanUseToolFn 判断 allow / deny / ask。不同运行模式使用不同的权限处理策略（交互式、协调者、Worker）。

5

钩子 — Pre/Post Hooks

执行前后触发用户自定义钩子脚本。钩子可以修改参数、中止执行或记录结果，实现非侵入式扩展。

6

执行 — Execution

核心调度逻辑：只读工具可并行批量执行，写入工具串行执行。通过 isConcurrencySafe 属性分区调度。

7

遥测 — Telemetry

完整的执行追踪：startToolSpan / endToolExecutionSpan，支持 Perfetto 格式和 BigQuery 导出。

04 — 权限系统

安全守门人

精细化的权限控制确保智能体在安全边界内运行。多层次的权限模式和可编程规则引擎提供灵活的安全策略。

🔐 权限模式 PermissionMode

  • default — 标准交互模式，需要用户确认危险操作
  • plan — 只读计划模式，禁止所有写入操作
  • bypassPermissions — 完全信任模式，跳过所有权限检查
  • 自定义模式 — 通过配置文件灵活定义

📋 权限规则 PermissionRule

  • 基于工具名称的 allow/deny 规则
  • 基于参数模式匹配的细粒度控制
  • CLAUDE.md 配置文件中的声明式规则
  • Settings / MDM 路径下的全局策略

⚡ 处理策略 Handlers

  • interactiveHandler — 向终端用户发起确认
  • coordinatorHandler — 由协调者统一决策
  • swarmWorkerHandler — Worker 模式下的受限权限
  • 桥接模式 — IDE 集成时的回调处理
05 — MCP 协议

开放的扩展协议

Model Context Protocol (MCP) 是 Claude Code 连接外部服务的标准协议。通过统一的客户端架构支持多种传输方式和能力暴露。

🔌 传输层 Transports

  • ⌨️
    Stdio Transport
    标准输入/输出，本地进程通信
  • 📡
    SSE Transport
    Server-Sent Events，单向流式传输
  • 🌐
    Streamable HTTP
    HTTP 流式传输，灵活部署
  • 🔗
    WebSocket Transport
    全双工通信，实时交互
  • ⚙️
    InProcess Transport
    进程内直接调用，零延迟

📦 能力暴露 Capabilities

  • 🔧
    Tools — 工具暴露
    MCP 服务器注册的工具自动合并到工具列表
  • 📄
    Resources — 资源读取
    ListMcpResources / ReadMcpResource 统一访问
  • 💬
    Prompts — 提示模板
    从 MCP 服务器加载结构化提示
  • 🔑
    OAuth — 认证
    McpAuthTool 处理 OAuth 认证流程
  • 🔔
    Elicitation — 信息获取
    服务器可主动向用户请求补充信息
06 — 工程体系

生产级基础设施

从构建工具到可观测性，Claude Code 的工程体系确保了代码质量、运行时稳定性和持续迭代能力。

⚡ 构建系统

基于 Bun 运行时和打包器。大量使用 feature() 宏实现编译时特性开关和死代码消除，按需加载模块。

Bun bun:bundle Feature Flags Dead Code Elimination

🔍 代码质量

双引擎 lint 策略：Biome 负责格式化和基础检查，ESLint 配合自定义规则处理复杂场景，确保一致的代码风格。

Biome ESLint Custom Rules TypeScript Strict

🚨 错误处理

分层错误体系：ClaudeError 基类、AbortError 中断、ShellError 命令错误、TelemetrySafeError 安全上报。工具错误通过 classifyToolError 智能分类。

ClaudeError AbortError ShellError classifyToolError

📊 可观测性

完整遥测管线：事件上报、会话追踪、工具执行 Span。支持 Perfetto 格式可视化和 BigQuery 数据导出，插件也有独立遥测通道。

Perfetto BigQuery Session Tracing logEvent

🧩 模块系统

严格的模块边界和职责划分。每个工具/命令一个目录，服务层独立封装。支持条件导入和惰性加载，最小化启动开销。

Modular Directory Lazy Loading Barrel Exports

🔄 数据迁移

内置迁移机制处理配置和数据的版本升级。专用 migrations/ 目录确保向后兼容性，平滑的版本过渡体验。

migrations/ Settings Validation MDM Paths
07 — 查询管线

推理循环的心跳

查询管线是智能体的推理引擎核心。它管理着从用户输入到模型响应、工具调用到结果整合的完整循环。

入口

消息构建

接收用户输入，构建结构化消息对象。包括文本、附件、上下文信息。消息经过规范化后进入查询队列。

utils/messages.ts types/message.ts
预处理

Token 预算 & 压缩

计算当前对话的 Token 用量，必要时触发消息压缩（compact）——智能截断历史上下文，保留关键信息。

services/compact/ cost-tracker.ts
API 调用

Claude API 交互

向 Claude API 发送请求，处理流式响应。包含重试逻辑、错误处理、用量统计和中断支持。

services/api/ query.ts
工具执行

并发编排 & 执行

解析模型返回的工具调用请求，通过 partitionToolCalls 分区后并行或串行执行。使用 AsyncGenerator 实现流式结果返回。

循环
